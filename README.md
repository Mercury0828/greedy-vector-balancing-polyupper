# greedy-vector-balancing-polyupper

Research project for idea `greedy-vector-balancing-polyupper` (target venue: **SODA**).

**Goal.** Prove that the deterministic greedy Euclidean vector-balancing algorithm of Czerwiński–Dadush–Ergen–Ghosh–Lasota–Orlikowski (arXiv:2606.17991) has a **polynomial** worst-case norm `G(T) = poly(d, 1/δ_T)` for finite `T ⊆ S^{d-1}` with constant (or `1/poly(d)`) distance property `δ_T` — exponentially improving their `(2/δ_T)^{d-1}` upper bound and resolving, on the polynomial side, the tightness question they leave open.

**How to run.** Open this directory in Claude Code and paste the contents of `start_up_for_claude_code.md` to begin Phase 0 (scaffold + literature kill-scan + adversarial de-risk of the load-bearing crux, stopping at a human gate). `guide.md` is the read-only project constitution (research-attack orchestration: external-solver attack loop, audit/rebattle, kill→pivot, honesty rails).
